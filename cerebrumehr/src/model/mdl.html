Query Sample

The query consists of a leading $ character followed context name, and or followed by a dot character (dot character is optional) and another identifier. These are examples of valid property references in the Magic Report
Author: NA
Company Name: No Magic
Project Name: model
Blank Value: NA
Total Package: 16
Use Case Owner:
There are three type of defined context, which is able to query by Magic Report
  1. Open API contexts – This context consists model elements that available on Open API or OMG meta-model. Ex. UseCase, Class, Package, and Diagram
  2. Domain Specific contexts – This context consists of domain specific model elements. The domain specific element is an extension of OMG meta-model (combining between element and stereotype). Ex. requirementUseCase
  3. User defined contexts – Use defined this context from graphical user interface. Ex. Author, CompanyName
  4. Helper contexts – This context consists of tools that provide additional features or convenience functions for create template. Ex. sorter, bookmark, report
Note: Any non-value in query will be replace with output on blank field options during generating report.

Escape Characters Sample

The query syntax of template is begin with $ (dollar sign) and followed by letter. But sometime you may need to use dollar sign as character in report. Magic Report provides escape character for this condition.
Example of dollar escape character:
  • Dollar Sign \$value
  • Dollar Sign \$5000
  • Dollar Sign $5000
Example of shape sign escape character:
  • Shape Sign \#
  • Shape Sign \#value
Note: Magic Report automatically removes paragraph and row at line or row of command statement
